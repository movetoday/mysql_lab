### **🚀 1개월 차: 기본기 재정비 및 심화 (Foundation Month)**

**목표**: 5년의 공백을 메우고, 어떤 기술 면접에서도 막힘없이 대답할 수 있는 탄탄한 기본기를 갖춥니다.

- **1주차: RDBMS 핵심 이론 다시 파고들기**
    - **📚 학습 내용**:
        - 트랜잭션 ACID, 격리 수준(Isolation Level) 별 차이점 및 발생 가능한 문제 (Dirty Read, Phantom Read 등)
        - B+Tree 인덱스 구조, 클러스터형/비클러스터형 인덱스 동작 원리
        - 정규화(Normalization)의 장단점 및 비정규화(Denormalization)가 필요한 경우
    - **💻 실습 과제**:
        1. **블로그 포스팅**: "MySQL(InnoDB)의 격리 수준 4가지를 예제 코드로 설명하기" 라는 주제로 글 작성하기. (개념을 명확히 설명하는 연습)
        2. 테이블을 직접 설계하며 1, 2, 3차 정규화를 순서대로 적용해보기.
- **2주차: 실무 SQL 마스터 및 실행 계획 분석**
    - **📚 학습 내용**:
        - 윈도우 함수(Window Functions: `ROW_NUMBER`, `RANK`, `LEAD`, `LAG` 등)
        - CTE(Common Table Expressions)를 활용한 가독성 높은 쿼리 작성법
        - `EXPLAIN`을 통한 실행 계획 분석 (key, rows, Extra 컬럼 집중 분석)
    - **💻 실습 과제**:
        1. LeetCode, HackerRank 등에서 SQL 문제 풀기 (Medium ~ Hard 난이도)
        2. **성능 저하 쿼리 튜닝**: 의도적으로 비효율적인 쿼리(ex. 인덱스 없는 컬럼으로 `WHERE`절 사용)를 작성한 후, `EXPLAIN`으로 분석하고 인덱스 추가 등을 통해 개선하는 전/후 과정 기록하기.
- **3주차: DBA를 위한 Linux**
    - **📚 학습 내용**:
        - DB 서버 성능 분석을 위한 명령어: `top`, `vmstat`, `iostat`, `netstat`
        - 파일 시스템 구조 및 권한 관리 (`chmod`, `chown`)
        - 주요 로그 파일 위치 및 `tail`, `grep`을 이용한 실시간 로그 분석
    - **💻 실습 과제**:
        1. WSL2 또는 VirtualBox에 Linux(Ubuntu) 환경 구축하고 MySQL 직접 설치하기.
        2. MySQL에 부하를 주면서(e.g., 벤치마크 툴 `sysbench` 사용) `top`으로 CPU, 메모리 변화 관찰하기.
- **4주차: Shell Script와 자동화 기초**
    - **📚 학습 내용**:
        - 변수, 조건문, 반복문을 활용한 간단한 Bash 스크립트 작성법
        - `cron`을 이용한 주기적인 작업 스케줄링
    - **💻 실습 과제**:
        1. **자동 백업 스크립트 작성**: `mysqldump`를 이용하여 특정 데이터베이스를 매일 새벽 2시에 자동으로 백업하고, `백업파일명_날짜.sql` 형태로 저장하는 쉘 스크립트를 작성하여 `crontab`에 등록하기.

### **🔧 2개월 차: 핵심 직무 역량 강화 (Core DBA Skills Month)**

**목표**: 실제 운영 환경에서 마주할 문제들을 해결하는 실전 능력을 기릅니다.

- **5주차: 성능 튜닝 실전**
    - **📚 학습 내용**:
        - 인덱스 전략: 커버링 인덱스, 복합 인덱스 컬럼 순서의 중요성
        - MySQL 주요 시스템 변수(Parameter)의 의미 (`innodb_buffer_pool_size` 등)
    - **💻 실습 과제**:
        1. 대용량 테스트 데이터(100만 건 이상)를 생성 후, 특정 쿼리의 실행 계획 및 속도를 측정.
        2. 커버링 인덱스, 복합 인덱스 등을 적용하며 인덱스 종류에 따른 성능 변화를 직접 측정하고 블로그에 기록하기.
- **6주차: 백업 및 복구 시나리오 훈련**
    - **📚 학습 내용**:
        - 논리적 백업(Logical) vs 물리적 백업(Physical)
        - 바이너리 로그(Binary Log)를 이용한 특정 시점 복구(Point-in-Time Recovery, PITR)
    - **💻 실습 과제**:
        1. **장애 시뮬레이션**: `mysqldump`로 전체 백업을 받은 후, DB에 여러 건의 데이터 변경(INSERT, UPDATE, DELETE)을 수행.
        2. 실수로 특정 테이블을 `DROP` 한 상황을 가정하고, (1)의 전체 백업 파일과 바이너리 로그를 이용해 테이블이 삭제되기 직전 시점으로 복구하는 전체 과정 실습 및 기록.
- **7주차: 모니터링 및 장애 대응**
    - **📚 학습 내용**:
        - Slow Query Log 설정 및 분석 방법
        - `Performance_Schema`를 활용한 DB 내부 동작 분석
        - Deadlock 발생 원인 및 해결 방안
    - **💻 실습 과제**:
        1. Slow Query Log를 활성화하고, 일부러 느린 쿼리를 실행시켜 로그가 쌓이는 것을 확인. `pt-query-digest` 같은 툴로 리포트 생성해보기.
        2. 의도적으로 Deadlock을 유발하는 쿼리를 두 세션에서 동시에 실행하고, `SHOW ENGINE INNODB STATUS` 명령어로 원인 분석하기.
- **8주차: 데이터 모델링과 스키마 변경 관리**
    - **📚 학습 내용**:
        - ERD(Entity-Relationship Diagram) 설계 개념 복습
        - Flyway, Liquibase 같은 DB 마이그레이션 툴의 필요성과 동작 원리 이해
    - **💻 실습 과제**:
        1. 간단한 쇼핑몰(사용자, 상품, 주문 테이블) ERD를 직접 그려보고, `CREATE TABLE` 문 작성하기.
        2. Flyway를 로컬에 설치하고, 버전별로 SQL 파일을 관리하며 스키마를 변경(테이블 생성 -> 컬럼 추가 -> 인덱스 생성)하는 과정 실습하기.

### **1. 데이터베이스 이론 심화**

- **학습 방법:**
    - **개념 정리:** 전공 서적이나 신뢰도 있는 온라인 자료를 통해 핵심 개념을 다시 한번 꼼꼼하게 읽고, 자신만의 언어로 노트에 정리해 보세요.
    - **사례 연구:** 각 개념(정규화, 트랜잭션 등)이 실제 서비스에서는 어떻게 적용되는지, 특정 선택(예: 비정규화)을 했을 때 얻는 이점과 감수해야 할 단점은 무엇인지 구체적인 사례를 찾아보세요.
- **추천 자료:**
    - **📖 도서:**
        - **"데이터베이스 시스템 (일명 '공룡책')"**: 가장 정석적인 교재입니다. 필요한 챕터(정규화, 트랜잭션, 동시성 제어 등)를 발췌해서 다시 한번 정독하는 것을 추천합니다.
        - **"Real MySQL 8.0" (1권)**: MySQL을 기준으로 설명하지만, 데이터베이스의 전반적인 아키텍처와 핵심 원리를 매우 깊이 있게 다룹니다. 특히 4장(트랜잭션과 잠금), 5장(MySQL 아키텍처)은 필독입니다.
    - **💻 온라인 강의/자료:**
        - **K-MOOC "데이터베이스 시스템"**: 대학 강의 수준의 체계적인 학습이 가능합니다. ([http://www.kmooc.kr/](http://www.kmooc.kr/))
        - **쉬운코드 "데이터베이스 개론" 유튜브 재생목록**: 핵심 개념을 시각적으로 쉽게 설명해 주어 리마인드하기 좋습니다.
